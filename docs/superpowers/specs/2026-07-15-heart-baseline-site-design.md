# Heart Baseline Site Design

## Approach

Add Heart Baseline as a coming-soon app using the site's existing static HTML/CSS architecture. Reuse the shared product-page layout and give the app a dark charcoal theme with a restrained red accent. Use the four supplied marketing screenshots as the product story.

## Pages and files

- Add a Heart Baseline card, metadata, and footer links to the root page.
- Add `/heart-baseline/` with a hero, feature summary, four-image story, and support call to action.
- Add `/heart-baseline/privacy-policy/` covering local history storage, Apple Health heart-rate reads, paired-device Watch Connectivity transfers, deletion, and the absence of developer analytics, advertising, accounts, or servers.
- Add Heart Baseline to the shared support selector, subject prefix, return routing, and support tests.
- Copy the supplied screenshots into `/heart-baseline/assets/` with stable web filenames.
- Update the sitemap and README route list.

## Key rules

- Show Heart Baseline as `Coming soon`; do not add an App Store download link yet.
- Keep claims informational and non-diagnostic. State that Heart Baseline is not a medical device and does not provide medical advice.
- Preserve the existing responsive, accessible, dependency-free site structure.
- Include explicit image dimensions and useful alt text.
- Validate internal links, assets, metadata, support behavior, and responsive rendering before completion.
